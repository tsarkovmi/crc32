# crc32
Cyclic redundancy check - Циклический избыточный код.

Является совершенным (по сравнению с контрольными суммами) способом проверки достоверности данных.
Алгоритм CRC основан на полиномиальной арифметике. Делимое, делитель и частное рассматривается не как целые числа, а как полиномы с двоичными коэффициентами. 
Число записывается в виде двоичной строки, биты которой служат коэффициентами полинома.
Вычисление CRC - ни что иное как взятие остатка от деления. Для этого в первую очередь выбирается делитель.
Его длина должна превышать длину итогового CRC на один разряд, то есть для получения CRC16 необходим 17-битный делитель. Это связано с тем, что остаток должен быть меньше делителя, иначе можно выполнить деление еще один раз. Не все делители дают хорошие результаты, поэтому обычно выбирают так называемые стандартные делители.
В данном случае используется делитель F17D91F8.

На вход подается текстовый файл input.txt, в стандартный поток выводится полученный CRC32 для делителя F17D91F8.
